Golang一些有趣的工具

Mar 8th, 2015 12:24 am

最近发现了不少Golang的包，随手记一些我感兴趣的，并且更新吧。这样会轻松很多 ps：基本都是还未尝试，评价只是一种自我感觉。另外，我是个菜鸡，在类似的库间选择时难免有一些比较搞笑的评判标准。勿喷~~~

web框架

  • net/http 包，最最好用的小项目包没有之一。写一个小项目，没有那么复杂的逻辑，标准包就够用了。大项目，感觉还是有点薄弱。比如路由、参数解析等等。都需要自己研究，虽然说可以使用基本包实现，但是像我这么懒的，既然有现成的，为什么不用呢？
  • gin 对于net/http增强，简单、灵活不简单。貌似比Martini性能更好
  • revel 和 beego 都是基于MVC的框架, 都没有仔细读过。个人比较偏向于revel, 比较像rails。

包管理

  • gopm 类似于ruby的gem，最主要的是它解决了我对于多个项目GOPATH应用的。他的配置我没有搞懂。。。
  • godep 和gopm类似，不过它的‘Gemfile’是一个json。
  • gom 最终选择了这个，因为它首先可以在每个项目使用独立的库管理。另外，我可以把它当做go工具来用。也支持lock
  • gpm 非常轻量级，使用起来简单。他就管下载不同版本的包，使用不同版本的包。配个gvp使用，可以管理GOPATH。但是操作有点复杂，并且在我zsh下gvp还会挂。。。算了

数据库对象映射模型

  • xorm 目前知道这个

路由

  • mux 非常轻量的一个 ‘HTTP request multiplexer’ 配合net/http使用也挺不错。

另外还有redis和mysql的各种包，因为已经有很专业的了，我就根据官方推荐选择啦~~~

暂时就这样，知道了新的，或者我改变了主意。还会更新这个文章的。。。。

Golang 黑魔法之一: 使用strings.Split()函数时，如果string不包含sep则返回[string]而不是[]

Feb 16th, 2015 10:43 am

学习golang一小段时间了，深深被其敢于改变的特性所吸引。但有时也不免改动过大，让人难以理解

这是一个随手笔记，可能会有错误。。。。。

直接上代码

package main

import (
    "fmt"
    "strings"
)

func main() {
    fmt.Println(strings.Split("", "ss"))
    fmt.Println(strings.Split("a", "ss"))
}

它将有如下输出

[""]
["a"]

所以在golang中认为split失败时需要返回一个Array，这个Array的元素是其本身。其他语言(至少在我的认识中)则认为返回一个空Array。这是对我而言一个坑。因为如果这样使用，会有问题

package main

import (
    "fmt"
    "strings"
)

func main() {
    fmt.Println(strings.Split("", "ss"))
    for _, word := range strings.Split("a", "ss") {
        fmt.Println(word)
    }
}

我希望它是输入(所有的双引号只是用来区分字符串长度什么的，真正的输出是不会有双引号的)

0
"" //空字符串

但它的输出会是这个样子

1
"a"

所以这个坑暂时没有解决办法，有想法的可以交流下~~~
