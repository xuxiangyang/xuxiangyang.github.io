0.现代前端基础常识

Feb 19th, 2016 4:21 pm

npm: 相当于ruby中的bundle，利用package.json管理

但它的功能更强大，除了包含作者等等基本信息，还用scripts定义了可以执行的脚本。dependencies是你的依赖，就像gem的依赖。devDependencies是你开发时用的工具，里面的东西和ruby中类似于pry等gem相当。几个常用的命令：

npm负责你程序的脚本，依赖、下载。所以看一个前端，第一个应该看它的package.json

webpack: 打包工具

它可以将我们各种js、css、图片。利用各种loader加载好。这样，我们需要任何一个资源（不管是js、css、图片）都可以使用require来加载了。 它依赖于webpack.config.js。里面会定义各种各样的loader和plugin。之所以可以require各种资源，是因为给他们定义了对应的loader。 它有一个webpack-dev-server可以用来做开发是起web服务

babel: 用来转义js，jsx、ecma6、ecma7都由它转义

js出了各种写法，比如jsx、ecma6、ecma7等等。但很多浏览器不支持这种写法，所以需要babel来进行转义，转义成浏览器支持的形式。babel分成了好多个项目。要注意分清楚。

JSX: 就是js、html混用，写法类似下面的

但jsx本身是不被浏览器支持的。他需要babel进行转义。 他有一些不同

  1. class属性变为了className，因为class与js语法冲突
  2. 如果往原生HTML中加入HTML规范中不存在的属性，需要使用data-前缀

这个东西可以用sublime package Babel高亮。每个项目下有一个m.sublime-project。以后的输入用subl m.sublime-project代替subl . 会更合理。直接使用这个项目的特定sublime配置

ECMA6/ECMA7: 这是js的新规范。里面定义了一些新的语法。同样目前不能被所有浏览器直接执行。需要用babel进行转义。

数据绑定

现在很多库推崇的都是不要直接操作DOM，而是操作虚拟DOM，然后由库来决定真实DOM要不要重新渲染。常见形式有单向绑定和双向绑定。

  • 单向绑定: 可以通过改变数据改变DOM，不可以通过改变DOM改变数据。react就是这种方式。整体感觉就是如果想改变DOM，那么就去改变数据。如果数据不变，DOM其实是也不能改变的。这种方式效率高，模型简单。我们只需要考虑数据的改变就好了，但实际使用中form的input会有个小坑（form中的input一般都绑定在一个数据上，如果输入，其实是改变了DOM，数据并没有改变。所以表现出来的现象是不能输入和删除）。
  • 双向绑定：通过改变数据改变DOM，也可以通过改变DOM改变数据。angular采用这种方式。这种方式其实写小应用挺好用，数据和DOM相当于是同一个东西，不会存在单向绑定中form的input的那种问题。但是效率低，并且如果应用复杂后，其实是引入了很多不确定性。

依赖注入

angular中着重讲的概念。其实就是函数式编程。在js中函数也是对象，函数可以和其他对象一样可以当参数传递。所以会见到很多这种写法

使用的时候就是把声明好的函数传进去。在angular中，一般用这种方式把http、route等函数传进去。同样react也会用这种方式，传递一些函数。比如flux/redux中的dispatch、getState等
